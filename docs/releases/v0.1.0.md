# RaySkillHub v0.1.0

## Summary

- `RaySkillHub` 现在明确定位为 `RayHub` 的 skill 互通子项目，而不是整个产品本身。
- 建立了 `RaySkillHub` 作为本地 shared control plane 的基础能力：discovery、registry、state、diagnostics、sync、run。
- 同时覆盖两类使用者：Human CLI 用户，以及通过 `--json` 消费同一控制面的 OpenClaw / agent 调用方。
- 在 `v0` 阶段把 CLI contract、`sync`/`run` 语义、OpenClaw 决策文档和发布资料补齐到可实际落地使用的程度。

## Highlights

### Human CLI

- 提供 `list / inspect / doctor / sync / init-skill / run` 全套本地命令。
- 输出增强为更适合人工排障的形式，例如：
  - `list` 显示安装覆盖率、缺失 agent、`exec` 能力
  - `inspect` 显示 `hasRunScript` / `canExec`
  - `sync` 显示 `no-op`
  - `run` 显示 `exec_requested` / `executed`
- 修复了多个真实 CLI 参数解析问题，减少 `--json`、位置参数和选项值之间的歧义。

### Agent / OpenClaw

- 所有主要命令支持稳定的 `--json` envelope：
  - `contractVersion`
  - `command`
  - `status`
  - `operation`
  - `writeIntent`
  - `exitCode`
  - `output`
  - `data`
- `doctor --json` 增加结构化 summary：
  - `issueCounts`
  - `codeGroups`
  - `topRiskCodes`
  - `riskLevel`
- `sync --json` 增加：
  - `executionIntent`
  - `noOp`
  - 冲突修复预览/执行信息
- `run --json` 增加：
  - `execRequested`
  - `executed`
  - 错误路径结构化 `data.error`
- `codex` adapter 现在会为目标 `SKILL.md` 生成 Codex-native YAML frontmatter
- `claude-code` adapter 现在会为目标 `SKILL.md` 生成 Claude Code-compatible YAML frontmatter

### Docs

  - `docs/human-cli-guide.md`

## Notable Changes

- `list`:
  - 显示安装覆盖率与缺失 agent
  - 暴露 `hasRunScript` / `canExec`
- `inspect`:
  - 提供更完整的安装状态、历史和执行能力位
- `doctor`:
  - 提供结构化风险摘要，适合 agent triage
- `sync`:
  - 支持 targeted sync
  - 支持 `--repair-conflicts`
  - `no_source_changes` 统一为成功 no-op 语义
- `run`:
  - 支持准备与执行两种模式
  - 通过 `execRequested` / `executed` 明确表达执行状态
  - 错误路径返回结构化数据
- `docs`:
  - 建立从使用、测试、发布到 release 文案的完整资料链路

## Compatibility Notes

- No breaking changes in documented `v0` command contracts.
- Added backward-compatible fields including:
  - `operation`
  - `writeIntent`
  - `summary.codeGroups`
  - `summary.topRiskCodes`
  - `sync.executionIntent`
  - `sync.noOp`
  - `inspect.hasRunScript`
  - `inspect.canExec`
  - `list.hasRunScript`
  - `list.canExec`
  - `run.execRequested`
  - `run.executed`
- Agent consumers should prefer structured `data` fields over parsing `output` text.

## Upgrade Guidance

如果你是人类 CLI 用户：

- 建议先跑 `node dist/cli/index.js doctor`
- 涉及同步时先跑 `sync --dry-run`
- 若要执行 skill，先看 `inspect` 的 `can exec`

如果你是 OpenClaw / agent 调用方：

- 继续优先消费 `data`
- 对 `sync` 增量接入 `noOp`
- 对 `run` 增量接入 `execRequested` / `executed` / `error`
- 不要依赖 `output` 文本做主决策

## Known Limitations

- 当前仓库仍未包含 GitHub Actions 发布流水线
- 当前仓库仍未包含 npm publish 流程
- 仅覆盖仓库内已声明 adapter 与本地控制面能力
- `codex exec` 已验证可消费带 Codex-native frontmatter 的 skill；但如果 `~/.agents/skills` 中残留旧格式 skill，Codex 仍可能输出 loader 错误
